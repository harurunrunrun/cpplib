## API

- `quickhull_3d(points)`: 点集合の三次元凸包を、外向き三角形面からなる
  `ConvexPolyhedron3` として返す。各面は隣接面、担当する外側点集合、および
  最遠点を保持する。外側点を持つ候補面を保持し、展開点から見える
  連結な面領域だけを隣接関係に沿って探索する。削除面が担当していた点だけを
  新しい horizon cone へ再配置するため、展開ごとの「全生存面 x 全入力点」
  走査は行わない。

重複点は一つにまとめる。入力が空、同一点のみ、共線、または共面の場合も、
`affine_dimension` がそれぞれ `-1`, `0`, `1`, `2` の退化凸包を返す。

## API別の時間計算量・空間計算量

- 点数を $N$、出力面数を $K$ とする。座標順の整列に $O(N\log N)$ を使い、
  conflict graph 付き三次元 QuickHull の期待時間は $O(N\log N+K)$、
  標準的な最悪時間は $O(N^2)$。
- 生成途中の削除済み面を含む面数を $T$ とすると、領域は $O(N+T)$。
  通常は $O(N+K)$、最悪は $O(N^2)$。

各点は同時には一面の outside set だけに所属する。面の展開では、その可視領域の
outside set の和集合だけが再判定対象になる。面の隣接辺の照合はハッシュ表の
期待 $O(1)$、候補面の追加と取り出しは償却 $O(1)$ である。

## 注意点

- 可視性判定には適応的 `orient3d` を使う。
- NaN または無限大を含む点があれば `std::invalid_argument` を送出する。
- 入力は値渡しであり、呼び出し元の点列は変更しない。
- 点を辞書順に整列し、距離同値時は面番号・点番号で tie-break するため、同じ入力
  に対する頂点列と面列は決定的である。
